# KANBEI Sync — AI学習機能 完全ガイド

## 概要

KANBEI Syncは、経営者に寄り添うAI経理社員を実現するための学習エンジンです。
4層アーキテクチャで構成され、使うほどに経営者を理解し、パーソナライズされた経理サポートを提供します。

---

## 全機能一覧

### 1. 会社プロファイル（第1層 — 永続）

| 項目 | 内容 |
|------|------|
| 目的 | 会社の基本的な経理特性をAIが記憶する |
| 学習方法 | チャットで会社の情報を話すと自動学習 |
| 保存先 | `company_profiles` テーブル |
| チャットへの影響 | 毎回のチャットでシステムプロンプトに注入。回答の前提条件となる |

**学習される5つの項目：**

| フィールド | 内容 | 回答への影響 |
|-----------|------|-------------|
| 業種・事業内容 | IT、飲食、建設など | 勘定科目の判定基準が変わる |
| 経理の特徴 | 締め日、支払サイト、発生主義/現金主義など | 売掛金・買掛金のアドバイスが変わる |
| 特殊ルール | 会社固有の仕訳パターン、例外処理 | 仕訳提案が会社のルールに沿う |
| 税務メモ | 課税/免税、本則/簡易課税、インボイス対応 | 消費税計算の方法が根本的に変わる |
| その他 | 上記に当てはまらない重要情報 | 状況に応じた回答調整 |

**自動学習のトリガーキーワード：**
> 業種、事業内容、うちの会社、弊社、当社、自社、いつも、毎月、毎回、基本的に、原則として、特殊、例外、ルール、決まり、簡易課税、免税、課税事業者、インボイス登録

**編集UI：** `/company-profile` で確認・修正・クリアが可能

---

### 2. 経営者タイプ診断（第1層 — 永続）

| 項目 | 内容 |
|------|------|
| 目的 | 経営者のスタイルを理解し、コミュニケーションのトーンを最適化する |
| 診断方法 | 4軸 × 3問 = 12問の2択診断（多数決判定） |
| 保存先 | `owner_profiles` テーブル |
| チャットへの影響 | 経営者タイプに合わせた報告・提案のトーンが変わる |

**4つの軸：**

| 軸 | 選択肢 | 影響 |
|---|--------|------|
| 事業方針 | 拡大型(E) vs 安定型(S) | 成長提案 vs 安定重視の提案 |
| 意思決定 | 直感型(N) vs 分析型(A) | 簡潔な結論 vs データ付き説明 |
| 資金方針 | 投資型(I) vs 堅実型(C) | 投資提案 vs コスト削減提案 |
| 組織運営 | 共創型(T) vs 独立型(P) | チーム視点 vs 経営者個人視点 |

**16タイプ例：**
- ENIT（革新開拓型）→ ビジョンに共感しつつ数字で現実を見せる
- SAIC（守護経営型）→ リスクと安全性を中心にした報告
- SACP（石橋経営型）→ 安心材料を前面に出した報告

**診断ページ：** `/owner-diagnosis`
**再診断：** いつでも「もう一度診断する」ボタンで可能

---

### 3. 年次要約（第2層 — 永続）

| 項目 | 内容 |
|------|------|
| 目的 | 決算期の対応記録と年間実績を永続保持し、翌年以降の対応に活かす |
| 生成方法 | 手動トリガー（年度を選択して「年次要約を生成」） |
| 生成元 | 財務データ（月別P/L、費用科目別集計）+ チャット履歴 |
| 保存先 | `annual_summaries` テーブル（年度ごとに1レコード） |
| チャットへの影響 | 直近3年度分をシステムプロンプトに注入。「去年はどうだった？」に答えられる |

**要約に含まれる4つの観点：**
1. **財務実績** — 年間の売上・費用・利益の推移と特徴
2. **チャットからの学び** — 重要な対応・判断の記録
3. **重要決定事項** — 経営・経理上の決定、例外処理の記録
4. **経営者タイプ評価** — タイプの強みが活きた場面、注意点が顕在化した場面

**ページ：** `/annual-summary`

---

### 4. 月次要約（第3層 — ローリング）

| 項目 | 内容 |
|------|------|
| 目的 | 直近の経理状況と継続課題を把握し、フォローアップする |
| 生成方法 | 手動トリガー（月を選択して「月次要約を生成」） |
| 生成元 | 月次財務データ + チャット履歴 |
| 保存先 | `monthly_summaries` テーブル |
| チャットへの影響 | 直近3ヶ月分をシステムプロンプトに注入 |

**ローリング圧縮の仕組み：**

| 経過期間 | 詳細度 | 内容 |
|---------|--------|------|
| 当月〜1ヶ月前 | full（詳細） | 各項目3-5文で具体的に記述 |
| 2ヶ月前 | condensed（要約） | 各項目1-2文に圧縮 |
| 3ヶ月以上前 | minimal（概要） | 各項目1文以内、核心のみ |

**ページ：** `/monthly-summary`

---

### 5. フォローアップ発言（第4層 — セッション）

| 項目 | 内容 |
|------|------|
| 目的 | チャットを開いた時に「寄り添い感」を演出する |
| 生成方法 | チャットページを開く or 新しい会話を始めるたびに自動生成 |
| 生成元 | 期日超過の売掛金、月次アクションアイテム、前回チャットからの日数、銀行残高 |
| チャットへの影響 | 冒頭の挨拶メッセージがパーソナライズされる |

**フォローアップの例：**
- 「期日超過の売掛金が2件あります。回収状況はいかがですか？」
- 「先月のアクションアイテムだった広告費の見直し、進んでいますか？」
- 「前回のチャットから5日経ちましたが、月末の締め作業は順調ですか？」

---

### 6. フィードバック学習（既存機能）

| 項目 | 内容 |
|------|------|
| 目的 | ユーザーの評価から回答品質を改善する |
| 方法 | チャットの回答に対して👍/👎 + 修正コメント |
| 保存先 | `chat_feedback` テーブル |
| チャットへの影響 | 同じトピックの質問時に過去のフィードバックを参照 |

---

### 7. リアルタイムDBコンテキスト（既存機能）

| 項目 | 内容 |
|------|------|
| 目的 | 最新の経理データに基づいた回答を提供する |
| 生成方法 | 毎回のチャットで自動取得 |
| 取得データ | 月別P/L（12ヶ月）、費用科目別集計、B/S、未回収売掛金、銀行残高・バーンレート |

## チャット時のシステムプロンプト構成

チャットの際、以下の順序でコンテキストがClaudeに渡されます：

```
1. ベースプロンプト（経理プロのペルソナ）
2. リアルタイム財務データ
3. 会社プロファイル ← KANBEI Sync
4. 経営者タイプ診断 ← KANBEI Sync
5. 年次要約（直近3年度）← KANBEI Sync
6. 月次要約（直近3ヶ月）← KANBEI Sync
7. フィードバック学習
8. 直近7件の会話履歴
```

## ユーザー向け：プロンプトのコツ

### KANBEIに会社を理解してもらうために話すべきこと

**初回〜利用開始1週間で伝えると効果的：**

```
「当社はWeb制作の会社で、主にBtoBで法人サイトを受注しています」
「売上は月末締め翌月末払いが基本です」
「簡易課税を選択しています」
「従業員は5名で、役員報酬も支払っています」
「外注費が毎月発生するので、勘定科目は外注費を使っています」
「交際費と会議費の区別は1人5,000円で分けています」
```

**キーフレーズ：** 「当社は」「弊社では」「基本的に」「毎月」「いつも」を文頭に使うと自動学習が作動します。

### 回答精度を上げるための質問例

```
「先月と比べて経費は増えた？減った？」
→ リアルタイムデータを参照して具体的な数字で回答

「今期の着地予想を教えて」
→ 月別P/Lトレンドから予測

「この領収書の勘定科目は？タクシー代で顧客訪問です」
→ 会社プロファイル（業種・ルール）を加味して判定

「資金繰りは大丈夫？」
→ 銀行残高・バーンレート・未回収売掛金から総合判断
```

### フィードバックの活用

**👍を押すべき場面：**
- 勘定科目の判定が正しかった時
- 仕訳例が自社のルールに合っていた時
- 資金繰りのアドバイスが的確だった時

**👎＋修正コメントを入れるべき場面：**
- 「うちでは交際費ではなく会議費を使います」
- 「簡易課税なので仕入税額控除の計算は不要です」
- 「この取引は課税ではなく非課税です」

→ 修正コメントは次回以降の同トピックの回答に反映されます。

## KANBEI Sync パネル（チャット画面右側）

チャット画面の右側に常時表示される4タブのパネルです。

| タブ | 表示内容 | アクション |
|-----|---------|-----------|
| 経営者タイプ | 診断結果（タイプ名・強み・注意点・KANBEI対応方針） | 未診断なら「診断を受ける」ボタン |
| 会社プロファイル | AIが学習した会社情報 | 「編集・詳細を見る」で編集ページへ |
| 月次要約 | 直近3ヶ月の要約 | 「すべて見る」で一覧ページへ |
| 年次要約 | 直近の年度要約 | 「すべて見る」で一覧ページへ |

ヘッダーの「Sync」ボタンで開閉できます。

## オンボーディング（初回ユーザー向け）

新規ユーザーが初回ログインすると自動的にオンボーディングが開始されます。

**フロー：**
1. **ウェルカム画面** — KANBEIの紹介
2. **自社情報の登録** — 会社名（必須）＋ 決算月
3. **経営者タイプ診断** — 12問の2択
4. **完了画面** — チャットへ誘導
