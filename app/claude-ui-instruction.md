# Claude Code向け UI改善指示書

## 目的

添付スクリーンショットのReact製チャットUIを、**古く見える・のっぺりして見える・安っぽく見える**状態から、**現代的で落ち着いたSaaS UI**に改善してください。

対象は「AI経理社員」のチャット画面です。機能追加よりも、まずは**見た目の質感・余白・階層・一貫性**を改善することを優先してください。

---

## 現状の課題

以下が主な問題です。

### 1. 全体がのっぺりしている

* 背景・カード・チャット領域のトーン差が弱く、画面全体が平坦に見える。
* 情報のレイヤー分けが不足している。

### 2. チャット画面の主役が弱い

* 入力欄が目立たず、画面上の主要アクションに見えない。
* AIメッセージとユーザーメッセージの視覚的な強弱が弱い。

### 3. サイドバーが少し古く見える

* 濃いグラデーションや強い青ハイライトが重く見える。
* “業務アプリ感”はあるが、“洗練されたSaaS感”が足りない。

### 4. クイックアクションボタンがチープ

* ピル型の枠線ボタン + 絵文字の組み合わせが少し安っぽく見える。
* 情報密度と余白のバランスが悪い。

### 5. 余白ルールが揃っていない

* 広い場所と狭い場所が混在し、レイアウトのリズムが不安定。
* メッセージ間、セクション間、ボタン間の spacing に統一感がない。

---

## デザイン方針

以下の方向性でUIを改善してください。

### ゴール

* ChatGPT / Notion / Linear / modern B2B SaaS に近い、**静かで整った見た目**
* 派手さよりも、**上質さ・読みやすさ・信頼感**を優先
* 経理・会計領域のサービスにふさわしい、落ち着いたトーン

### キーワード

* minimal
* clean
* soft contrast
* clear hierarchy
* consistent spacing
* modern SaaS UI

---

## 実装方針

### 1. 背景レイヤーを整理する

* アプリ全体の背景は薄いグレー系にする
* メインコンテンツ領域は白ベースにして、面の違いを明確にする
* 「ページ背景」「サイドバー」「チャット面」「カード」のレイヤー差を作る

#### 例

* app background: `#f6f7f9`
* main panel: `#ffffff`
* subtle border: `#e5e7eb`

強い影より、**薄い border / ごく弱い shadow**を優先してください。

---

### 2. サイドバーをフラットで上品にする

* 現在の重い見た目を軽くする
* 濃いグラデーションはやめるか、かなり弱くする
* active state はベタ塗りではなく、薄い青背景にする
* アイコンとテキストの alignment を統一する
* セクション見出し（メイン / 受発注 / 会計）は、主張しすぎないラベル風にする

#### サイドバーの desired style

* dark but flat
* subtle separators
* active item: soft blue tint
* rounded corners: medium
* hover: mild brightness change

---

### 3. タイポグラフィを整理する

フォントサイズとウェイトの階層を明確にしてください。

#### 目安

* 画面タイトル: 24px / semibold
* セクション名: 12〜13px / medium / muted
* 本文: 14〜15px
* 補助テキスト: 12〜13px / muted

条件:

* 文字サイズの種類を増やしすぎない
* 日本語表示で読みやすい line-height を確保する
* 太字の使いすぎを避ける

---

### 4. チャットバブルを改善する

#### AIメッセージ

* 白 or ごく薄いグレー背景
* 枠線は薄く、影は極小
* カード感を出しすぎず、自然な面として見せる

#### ユーザーメッセージ

* 青系ベースでOKだが、今より少し深く落ち着いた色にする
* 真っ青すぎて軽く見えないようにする
* テキストの可読性を担保する

#### 共通

* 角丸は大きめでOKだが、過剰に丸くしすぎない
* メッセージ上下間隔を整理する
* 発話者ラベルは目立ちすぎないよう小さく muted にする

---

### 5. 入力欄を画面の主役にする

ここは重要です。入力欄が弱いので改善してください。

#### 改善内容

* 高さを少し増やす
* 角丸を大きくする
* 薄い border とごく弱い shadow を追加する
* プレースホルダの色を整理する
* 送信ボタンとの一体感を高める
* フォーカス時にきれいな ring を出す

期待する印象:

* 「ここに入力するUIだ」と瞬時にわかる
* 主張しすぎないが、操作起点として十分目立つ

---

### 6. クイックアクションを再設計する

現状のピル型ボタン + 絵文字はチープに見えやすいです。

以下のどちらかで改善してください。

#### 推奨案A

クイックアクションを残すが、**絵文字をやめてアイコンコンポーネントに置換**する。

* Lucide などの統一された line icon を使う
* 枠線だけのピルではなく、soft background のボタンにする
* 高さ、padding、gap を統一する

#### 推奨案B

クイックアクションを簡素化し、数を絞る。

* 4つ並べるならレイアウトを整える
* 小さいチップを乱雑に並べるのではなく、整列感を出す

要件:

* 安っぽく見えないこと
* 情報過多に見えないこと
* チャット本文より目立ちすぎないこと

---

### 7. 余白ルールを統一する

spacing system を導入してください。たとえば 4 / 8 / 12 / 16 / 24 / 32 ベースで統一します。

確認してほしい箇所:

* サイドバー内の項目間隔
* セクション見出しと項目の距離
* タイトルと本文の距離
* メッセージ間の距離
* クイックアクション同士の gap
* 入力欄まわりの余白

見た目の質は、色よりも spacing の整合性で大きく決まります。

---

### 8. コンポーネント単位でスタイルを統一する

以下の UI 要素で、radius / border / shadow / transition を統一してください。

対象:

* sidebar item
* chat bubble
* quick action button
* input field
* send button
* cards / panels

避けたい状態:

* 各コンポーネントで丸みや影のルールが違う
* hover/focus の挙動がバラバラ

---

## 技術方針

* React の構成は大きく壊しすぎなくてよい
* ただし UI の責務分離は必要なら進めること
* Tailwind を使っているなら、utility class を整理して再設計してよい
* shadcn/ui を導入済み or 導入可能なら、入力欄・ボタン・カード系は寄せてもよい
* ハードコードされた色・padding・radius が散っている場合は整理すること

可能なら以下を行ってください。

* デザイントークン化（color / radius / spacing / shadow）
* 再利用しやすい class / component 設計への寄せ

---

## 具体的にやってほしいこと

1. 現在のチャット画面UIを確認
2. スタイルの問題点を整理
3. レイアウトとコンポーネントの見た目を改善
4. 必要ならコンポーネントを分割
5. 改善後に、何をどう変えたかを要約

---

## 成果物の要件

以下を出してください。

### 必須

* 修正後コード
* 変更点の要約
* デザイン上の意図の説明

### あると望ましい

* 変更前に対して何が改善されたか
* 今後さらに良くするならどこか

---

## NG事項

* 派手なグラデーションを増やす
* 不必要に色数を増やす
* 影を強くしすぎる
* ボタンやカードを過剰に丸くする
* 絵文字ベースのUI表現を増やす
* “未来感”や“ネオン感”に寄せる
* 機能追加に走って、見た目の改善が中途半端になる

---

## 参考イメージ

目指すのは以下の中間です。

* ChatGPT の静かなUI
* Notion の整理された余白
* Linear のシャープな情報設計
* B2B SaaS の落ち着いた操作感

ただし丸コピーではなく、**経理業務ツールとして自然なデザイン**にしてください。

---

## 最終指示

見た目だけ少し変えるのではなく、**なぜダサく見えていたのかを解消するレベル**で調整してください。

特に重視する優先順位は以下です。

1. 余白
2. 階層
3. コントラスト
4. 一貫性
5. 質感

機能はそのままでいいので、**“素人が頑張って作ったUI” から “ちゃんと設計されたSaaS UI” に見える水準**まで持っていってください。
